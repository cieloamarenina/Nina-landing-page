Wie willst du Artikel schreiben & veröffentlichen?

Das bestimmt wie komplex das System wird

A

Einfach: Statisches HTML

Jeder Artikel ist eine eigene .html-Datei. Neuer Artikel = neue Datei anlegen, Link in blog.html eintragen. Kein Backend, kein CMS, 100% einfach. Perfekt für gelegentliche Posts.

B

Mittel: JSON-Datei als Mini-CMS

Alle Artikel in einer articles.json-Datei gepflegt. JavaScript lädt sie und baut Blog + Teaser dynamisch auf. Neuer Artikel = eine Zeile JSON. Kein Backend nötig.

C

Komplex: n8n / NocoDB Backend

Artikel werden in NocoDB gepflegt, n8n generiert daraus HTML-Seiten. Voll automatisiert, perfekt für häufige Posts — aber deutlich mehr Setup-Aufwand.
